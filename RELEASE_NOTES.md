# Release notes / publish checklist

Version: 1.0.0 (draft)

What changed
- Prepared README (English + Persian), added LICENSE (MIT template), SECURITY.md, and a short publishing checklist.

Before publishing
1. Confirm the upstream repository's license and ensure you have rights to publish derived code.
2. Optionally replace the `LICENSE` text if upstream requires a different license.
3. Attach `demo.gif` or a short recorded video to the GitHub release.
4. Tag the release: `git tag -a v1.0.0 -m "Initial public release"` and push tags.
5. Add repository `topics` (e.g., `chatgpt`, `tampermonkey`, `api`, `gapgpt`) on GitHub after publishing.

Publishing tips
- Write a short release description describing risks and that the repo is for educational use only.
- Share the release in Persian-speaking developer communities with the demo GIF and a short TL;DR in Persian.
